# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

Please **do not** report security vulnerabilities through public GitHub issues.

Instead, report them by emailing [jeffrey.limnardy@tum.de](mailto:jeffrey.limnardy@tum.de)
with the subject line `[SECURITY] <brief description>`.

You should receive a response within 48 hours. If you do not, please follow up to
ensure the original message was received.

Please include the following information:

- Type of issue (e.g. credential exposure, injection, path traversal)
- Full paths of the affected source file(s)
- Location of the affected code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

This information will help us triage your report more quickly.
